Basic Network and Database Security

2022-01-27

Completed IBM’s course Basic Network and Database Security on EdX.

Here are some of my main takeaways:

Stateless and Stateful Inspection

Stateless Inspection Stateful Inspection
Each packet inspected independently as no session table is maintained Each packet inspected with knowledge of previous packets in that same session as a firewall maintains a database
Faster, protecting routing engine resources If an incoming packet doesn’t match an existing session, the path through the firewall is longer
Examine source IP address, ACL rule, destination port Can see ingoing and outgoing interface
Helpful for routers with virtualization
Perform quality of service switches to prioritize traffic
OSI Model Layers Stateless Inspection Through the OSI Model Layers
(7) Application Open web browser
(4) Transport Browser creates TCP traffic, giving packet an IP address
(3) Network Packet header has source and destination IP address
(2) Data Link Physical or MAC address of source and destination computers, and MAC address of gateway added to packet header
(1) Physical Router evaluates packet before forwarding

Address Resolution Protocol

  • ARP takes IP address input and returns MAC address of that computer
  • MAC address of destination is only required when transmitting data within the same broadcast domain, otherwise only the MAC address of default gateway is needed
  • ARP table only translates addresses within the same local broadcast domain

IDS and IPS Systems

Firewalls have a:

  • Forwarding plane that forwards traffic, makes routing decisions, policy evaluations, and session matching
  • Control plane that runs the OS which controls the device and routing table

Both IDS and IPS examine network traffic to detect and prevent vulnerability exploits:

Intrusion Detection System (IDS) Intrusion Prevention System (IPS)
Monitors traffic and alerts administrator if needed, but doesn’t take action unless configured to run scripts Takes action automatically when an anomaly is detected
Dedicated server connected to a port on a switch Positioned after a router, edge device, or firewall
Switch forwards a copy of traffic to IDS (offline) All traffic passes through, which adds a delay to flow of network traffic (inline)
Anomaly-based detection: finds traffic that doesn’t follow the standards of its protocol Statistical anomaly-based detection
Signature-based detection: signatures describe common patterns of malware traffic
  • Other types of IDS and IPS systems:
    • Host-based IDS (HIDS): Software installed on endpoint machine to monitor traffic and may take action
    • Network-based IDS and IPS: Hardware, either configured to the core switch or sitting in the middle of the stream of traffic, that listens to traffic sent or flowing through it

Network Address Translation

Remaps IP address space by modifying the network address information in the packet headers when they’re passing through a NAT router

  • Increased security by hiding real IP addresses from the world
  • Shares one Internet-routable IP address of a NAT gateway for an entire private network
  • Types of NAT:
    • Static: one-to-one mapping between local and global addresses
    • Dynamic: maps unregistered to a registered IP address from a pool of them
    • Overloading: map all internal IP addresses to a single external/public one
      • NAT maintains database of addresses and port mapping

Ethernet and LAN- Ethernet Operations

OSI Model Layer
(3) Network Layer -Uses IP (IPv4 or IPv6) addresses -Data is encapsulated with a packet header, then another -Checks if the destination IP address matches its own and if it does, strips off layer 3 header and sends it to layer 4
(2) Data Link Layer -Header contains source and destination MAC IP addresses, protocol type, the data, and the checksum -Checks if the destination MAC addresses matches its own and if it does, strips off layer 2 header and sends it to layer 3
(1) Physical Layer -Ethernet cable or wireless connection connects computers to a LAN -Converts electronic signals to digital bits
  • Networks support full-duplex communications, so computers can send and receive data at the same time
  • Types of communication:
    • Unicast: one-to-one between two computers
    • Broadcast: one computer sends information to all computers on a network using the broadcast IP address
    • Multicast: one-to-many where endpoints subscribe to receive messages

Application and Transport Protocols TCP and UDP

In both TCP and UDP, the data stream is broken down into small chunks of data put into separate packets, addressed, and sent

Transport Control Protocol (TCP) User Datagram Protocol (UDP)
Sets up a connection to acknowledge all packets that have been sent and received in correct order Sender doesn’t know if packet has been received, and the receiver reassembles the data in the order the packets arrive
Slow Fast, good for streaming videos and music
Flow control (sent packets are not faster than receiving system) No flow control
Bundle of data and headers called a segment Bundle of data and headers called a datagram
Three way handshake to establish connection: send SYN to recipient, recipient responds with SYN ACK, sender responds ACK

DNS and DHCP

Domain Name Service (DNS) Dynamic Host Configuration Protocol (DHCP)
Translates domain names in URLs into IP addresses Automatically configures and manages IP addresses on endpoints from a pool of available IP addresses
Runs on local machine, often hosted on the default gateway Handshake consists of four packets between the requesting system and DHCP server: discover (broadcast message to all endpoints on network segment), offer, request, and acknowledgement
Marks IP address as leased to the MAC address of the endpoint

Clustering and High Availability Systems

  • Information security should ensure: Confidentiality, Integrity, and Availability
  • High availability is a system where a component is continuously operational to ensure optimal operational performance
  • Requirements for creating a HA cluster:
    • Hosts in a virtual server cluster must have access to the same shared storage with identical network configurations
    • DNS should be configured properly
    • Same OS and hardware
    • Connections between primary and secondary nodes
  • Three characteristics:
    • Redundancy: multiple components do the same task
    • Monitoring and Failover: secondary device should always be monitoring your primary device because it should be able to take over as primary role if needed
  • NIC teaming is when there is more than one NIC operating at the same time

Types of Data

  • Structured data: organized in a formatted repository so its elements are addressable for processing and analysis
    • Distributed databases and data warehouses are structured data
    • Flat-file databases take all information from all records and store everything in one table
      • Repetition, requires more storage
    • Relational databases separate information into numerous tables, linked through primary and foreign keys (primary key from one table that has been placed in another table)
  • Semi-structured data: not in a repository, but data has associated information like metadata that makes it more amenable to processing
    • Big data are semi-structured data
  • Unstructured data: in many different forms and isn’t typically good for mainstream relational databases
    • File shares are unstructured data

Securing Databases

  1. Discover
    1. Identify data sources and classify the type of data
    2. Entitlement reporting (who has access to the data or data source?)
    3. Vulnerability assessment (how secure is the repository?)
  2. Harden
    1. Reconfigure, mask, and encrypt data to protect it
  3. Monitor and protect
    1. Activity monitoring, blocking quarantine, and dynamic data masking
  4. Repeat

Where data sources are hosted:

  • On premises
    • Operate and have full control over the database
  • Infrastructure as a Service (IaaS)
    • Maintain full access to application, data, runtime, middleware, and OS
    • Provider owns, runs, and updates virtualization, networking, storage, and servers
  • Platform as a Service (PaaS)
    • Maintain full access to application and data
    • Provider owns, runs, and updates runtime, middleware, OS, virtualization, networking, storage, and servers
    • To secure PaaS, one could tokenize their data
  • Software as a Service (SaaS)
    • Provider handles everything; there is no ability to reconfigure any aspect

OS Command Injection

  • OS command injection is abuse of vulnerable application that causes execution of OS commands that are specified by the attacker
    • Vulnerabilities made possible by lack of input sanitization and by unsafe way developers execute OS commands
      • Need defensive code to verify a file parameter is correct
      • Attackers use semicolons to chain together multiple OS commands

For prevention:

  • Limit executing OS commands and instead use the functionality built into programming language or use third-party libraries to reduce attack surface
  • Run at the least possible privilege level, under a user account with the least required rights
  • Do not run commands through shell interpreters because this makes a lot of syntax, like the use of a semicolon, no longer valid
  • Use explicit paths when running executables
    • If a writeable folder is referenced in the path before the folder containing the valid executable, an attacker may install a malicious version of the application
  • If possible, do not let user input reach the place where you execute the command unchanged
    • Modify user input or replace user-specified values with generated IDs
      • For example, instead of letting users specify a particular file, have a numeric ID that corresponds to that file and when the parameters come in, that ID is passed in
  • Sanitize user input with strict whitelists (not blacklists)
    • Best to use the regular expression whitelist [A-Za-z0-9.]+

SQL Injection

  • Abuse of vulnerable application functionality that causes execution of attacker-specified SQL queries
  • Common types:
    • Error-based: tailor actions based on database errors that show how a system behaves and give back private information
    • UNION-based: adds union class to the query for data exfiltration
    • Blind injection: data can be inferred by executing many queries where behavior presents one of two outcomes
      • Boolean-based (one of two possible responses) or time-based (immediate vs delayed execution)
    • Out of band: trigger requests to other sites and exfiltrate data through a separate channel, like sending an HTTP request

Similar prevention tactics as OS injection:

  • Use prepared statements
    • Separate the query structure from query parameters
  • Sanitize user input
  • Do not expose native database errors to the user
    • Application errors should not expose internal information to the user; detailed error messages belong in internal log files
  • Limit database user permissions
  • Use ORM libraries
    • Create in-memory set of objects that map your database structure
